# /xpost — アドオン追加 X 投稿文ジェネレーター

このスキルは、本日 `docs/addons_data.json` に追加されたアドオンエントリを git diff から抽出し、X (Twitter) 投稿用のスレッド文を生成します。

## 実行手順

以下の手順を順番に実行してください：

### Step 1: 本日の追加エントリを抽出

```bash
git log --since="$(date +%Y-%m-%d) 00:00:00" --until="$(date +%Y-%m-%d) 23:59:59" --oneline
```

上記で表示されたコミットの中から `docs/addons_data.json` に追加されたエントリを取得：

```bash
git diff <コミットハッシュ>^..<コミットハッシュ> -- docs/addons_data.json | grep '^\+' | grep '"nameEn"'
```

各エントリの `nameEn`、`descriptionJa`（または `description`）、`category` を収集してください。

### Step 2: スレッド投稿文を生成

以下のルールに従ってスレッドを作成：

**構成（例: N件追加の場合）**

- **Hook（番号なし）**: 件数・カテゴリ傾向・「🧵👇」を入れる。ページ番号は付けない。
- **1/M〜(M-1)/M 中間ポスト**: カテゴリ別（AI画像系・動画系・タイマー系・素材系など）にグループ化して紹介。各アドオンは絵文字 + 名前 + 一言説明の形式。MはHookを除いたポスト総数。
- **M/M 最終ポスト**: まとめ + CTA（ポータルURL） + ハッシュタグ + Disclaimer

**Disclaimerテキスト（最終ポストに必ず入れる）**

```
※ 紹介内容はマーケットプレイスの説明文をもとに作成したものです。各アドオンの開発者・公式とは無関係です。実際の動作は各アドオンでご確認ください。
```

**ハッシュタグ（最終ポストのみ）**

```
#AdobeExpress #アドオン #AIツール #デザイン
```

### Step 3: 出力

各ポストをコードブロックで区切って表示してください。投稿文の文字数（X の上限 280文字）には注意し、超える場合は分割してください。

---

## 注意事項

- 件数が5件以下の場合はスレッドではなく単発ポストで構成を検討する
- `descriptionJa` が空の場合は `description`（英語）を要約して日本語で一言説明を作成する
- カテゴリが偏っている場合（全件 utility など）はグループ化よりも機能別にまとめる
- ポータルURLは末尾のユーザー指示に従う（指定がなければプレースホルダーのままにする）
