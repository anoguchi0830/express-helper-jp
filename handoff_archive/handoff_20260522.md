## 技術的知見

### Adobe Add-on の制約（実機確認済み）

| 方法 | 結果 | 理由 |
|------|------|------|
| `window.open()` | ❌ | COOP ヘッダー（ERR_BLOCKED_BY_RESPONSE） |
| `addOnUISdk.app.openBrowserUrl()` | ❌ | SDK に未実装 |
| `addOnUISdk.app.ui.openEditorPanel()` | ❌ | URL は変わるがパネルが開かない |
| **`document.execCommand('copy')`** | ✅ | ユーザー操作起点なら sandbox でも動作 |

### manifest.json の重要ルール

- `testId` は変更厳禁（アドオン ID が壊れる）
- `allow-clipboard-write` は manifest で許可不可
- Manifest V2 では `name` は単純文字列のみ（多言語化は Developer Console で実施）

### データ構造

- `addons_data.json`: 424件（addOnId 有 = 419件、なし = 5件）
- `marketplaceUrl` なし: `text-gradient-pro`, `undraw`, `tiktok-music`, `mockuuups`, `vcard-generator`
- `descriptionJa` / `descriptionKo`: 既存37件翻訳済み、新規382件は手動翻訳中

### CSS-in-JS の制約

- `border` shorthand は inline style 不可（transition が効かない）→ `borderWidth/Style/Color` ロングハンド必須
- `:hover` は inline style 不可 → `onMouseEnter/Leave` + `useState` で実装
